## 2. How did you use AI as a teammate?

- Which AI tools did you use on this project (for example: ChatGPT, Gemini, Copilot)?
- Give one example of an AI suggestion that was correct (including what the AI suggested and how you verified the result).
- Give one example of an AI suggestion that was incorrect or misleading (including what the AI suggested and how you verified the result).

I used Claude Code in VS Code. 
I tackled two problems:
  1. incorrect higher/lower hints
  2. incorrect number range

Correct suggestions:
It correctly refactored the higher/lower hints code into the logic utils and fixed the bug causing incorrect hints to be displayed. I verified this by analyzing the proposed change and running pytest tests and running the game.

Incorrect/missleading 
1. For the range fix, it assumed the range for "high" (1–50) was incorrect and needed a max of 500, and implemented it without flagging it as a logic change. I caught this during a proposed change review and assumed the higher difficulty comes from limiting the number of allowed attempts, not from increasing the number range.
2. Also for the range fix, it added logic defaulting the range to 1–100 if something other than easy/normal/hard was selected, even though the code doesn't allow other selections (it's a dropdown). I caught this during a proposed change review.

## 3. Debugging and testing your fixes

- How did you decide whether a bug was really fixed?
- Describe at least one test you ran (manual or using pytest)  
  and what it showed you about your code.
- Did AI help you design or understand any tests? How?

For the range issue, to verify that it was fixed, I carefully looked through the new code, asked the agent to generate tests, reviewed them, and ran them. I also played the game many times to make sure it behaves correctly.

The pytests generated by the agent caught my attention - I noticed they only check if get_range_for_difficulty returns the correct range, not that the range is correctly used in the game. I questioned the agent about this, and it explained that this logic lives inside app.py, mixed in with Streamlit display code and randomness, and runs as part of building a page. To make it testable, we'd need to pull the logic out into small, pure functions (e.g. one that builds the secret, one that builds the caption string) and test those. However, it suggested that for a game this small, that would be overengineering, which I agreed with - so I relied on the observed game behavior instead.

## 4. What did you learn about Streamlit and state?

- How would you explain Streamlit "reruns" and session state to a friend who has never used Streamlit?

Every time a user interacts with the app, Streamlit reruns the entire script from scratch, resetting any regular variables in the process. st.session_state is a persistent dictionary that survives these reruns, allowing us to maintain values across interactions.

I will keep working on code changes incrementally and verify all suggestions carefully to better follow all the proposals and changes performed by AI, as opposed to "fix all at once", approach where I'd lose visibility and understanding of the code, allowing undesirable results.

Next time I would set expectations and guidelines upfront - for example, "do not change or extend the existing logic".
